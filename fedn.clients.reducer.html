check_round_participation_policy(compute_plan, combiner_state)[source]¶

Evaluate reducer level policy for combiner round-paarticipation. This is a decision on ReducerControl level, additional checks applies on combiner level. Not all reducer control flows might need or want to use a participation policy.

check_round_start_policy(combiners)[source]¶

Check if the overall network state meets the policy to start a round.

check_round_validity_policy(combiners)[source]¶

At the end of the round, before committing a model to the model ledger, we check if a round validity policy has been met. This can involve e.g. asserting that a certain number of combiners have reported in an updated model, or that criteria on model performance have been met.

client_allocation_policy_first_available()[source]¶

Allocate client to the first available combiner in the combiner list. Packs one combiner full before filling up next combiner.

client_allocation_policy_least_packed()[source]¶

Allocate client to the available combiner with the smallest number of clients. Spreads clients evenly over all active combiners.
